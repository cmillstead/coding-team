# C14

| ID | Pattern | Check before dispatch |
|----|---------|----------------------|
| C14 | `@tags: command-grammar; provable; scope:both` **An ALLOW-LIST flag-hardening pass (drop an auto-approved command to ask when it carries a dangerous flag) fails along THREE independent completeness axes that a single-model audit under-covers: (1) COMMAND-set — the audit enumerates SOME allowlisted commands' dangerous flags but misses OTHER allowlisted commands that ALSO have exec/write/indirect-read flags; (2) FLAG-SPELLING — a value-taking SHORT flag guarded by exact-match misses the ATTACHED/BUNDLED form (`-oFILE`, `-rno out`); (3) INTERPRETER open-endedness — a command whose exec-flag surface is large and version-growing (node: -e/-p/-r/-i/--eval/--import/--loader/--experimental-loader/…) can NEVER be safely closed by a blocklist.** Caught over 6 Codex rounds on the Baxter 06-02 allowlist-dangerous-flag plan: round 1 the audit missed file/wc/du/pytest entirely (axis 1) and `has_flag` exact-match let `sort -oFILE`/`file -fL`/`pytest -pX` through (axis 2); rounds 2-5 kept finding new node exec flags one at a time until node was rewritten as a positive WHITELIST (allow only `node --check`/`-c` with no other option), then a positional edge (`node -- --check app.js` runs a script literally named `--check`) needed a stateful leading-option parse; go was under-audited and leaked `-o`/`-c`/`-overlay`/`env -w`/`-args` test-binary passthrough writes. Final: 19 concrete bypasses across 6 fix rounds, all same-model-audit-clean before Codex. | For any allow-list hardening (auto-approve → ask on a dangerous flag): **(axis 1)** enumerate EVERY allowlisted command, not just the obvious ones — for EACH, check its real man/help for exec, arbitrary-file-WRITE (`-o`/`--output`/compile/profile), and indirect-out-of-root-READ (`--files-from`/`--files0-from`/list-of-paths) flags; missing a whole command is the easiest gap. **(axis 2)** a value-taking SHORT flag must be matched by "single-dash cluster CONTAINS the letter" (catches `-oV`, bundled `-rno`), never exact `== "-o"`; a long flag needs both `--x` and `--x=` (cf. C10). **(axis 3)** if a command is an INTERPRETER or has an open-ended/version-growing exec-flag surface, do NOT blocklist its exec flags (whack-a-mole that loses on the next version) — use a positive WHITELIST: allow ONLY the one known-safe invocation shape, parsed STATEFULLY by argument position (options end at `--` or the first positional; a flag-looking token after that is a filename, not a flag). Direction must be fail-safe (allow→ask only) so every miss degrades to a human gate, never to silent allow. Pairs with C10 (flag-spelling completeness) and P33 (fail toward the safe state). |

**Design default:** For any allowlist hardening pass, enumerate every allowlisted command's dangerous flags (all three axes: full command set, cluster-matched short flags, interpreter → positive whitelist) before writing a single line of guard code.
