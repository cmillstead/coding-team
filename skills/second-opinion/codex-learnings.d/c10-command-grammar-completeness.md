# C10

| ID | Pattern | Check before dispatch |
|----|---------|----------------------|
| C10 | `@tags: command-grammar; provable; scope:both` **A security guard that ENUMERATES a command's option grammar to resolve a command-position (allow/deny) models only a SUBSET of each flag's spellings — short flag but not its long alias, or it omits a NON-EXEC mode — so a catastrophe behind the unmodeled spelling dilutes past the guard, and a non-exec mode mis-parses a benign trailing token into a false catastrophic argv0 (over-deny).** When a classifier hand-models a CLI's options to decide whether the rest is a wrapped command (exec-wrapper unwrap, sudo/privilege detection, an interpreter `-c` carve-out), modeling `-x` but not `--exclusive`, `-c` but not `--class`, `-w` but not `--wait` leaves the long form unhandled → the wrapped catastrophe falls to the ambiguous branch (here ASK, but in an allow-context could be silent). Symmetrically, treating a NON-EXEC selector (`chrt -p PID`, `ionice -p`, `chrt -m`/`--max` print-and-exit, `flock -c STRING`) as if it wrapped a command mis-binds a positional/trailing token as the inner argv0 → over-deny. Caught on the Baxter EXEC_WRAPPER unwrap: the first impl modeled short flags only; Codex found `flock --exclusive … rm -rf /`, `env --ignore-environment sudo reboot`, `setsid --wait`, `chrt --fifo`, `ionice --class`, `xargs -J` all diluting deny→ask, plus `chrt -m`/`ionice -p` over-denying. | For any guard that hand-models a command's option grammar, enumerate EACH modeled flag's COMPLETE spelling set — short `-x`, long `--alias`, `--long=value`, and combined `-xVALUE` — from the actual man page (don't model one spelling and assume the rest); a missed alias is a silent grammar hole. Separately, list the command's NON-EXEC / print-and-exit / operate-on-existing-PID / packed-string modes and route them to the SAFE-ambiguous state (ASK/None), never unwrap them as if a command follows. Verify the dominance property both ways: no catastrophe slips via an unmodeled common spelling (dilution), and no benign non-exec form newly denies (over-deny). Pairs with P33 (ambiguous→ask, not deny-on-guess) — C10 is the enumeration-completeness half: P33 says "don't guess," C10 says "model the whole real grammar so you don't have to." **Three spelling axes beyond short-vs-long that recur (Baxter 06-03, 3 Codex rounds): (1) CLUSTERED short-option bundles — a value/significant flag hidden mid-cluster (`bash -exc CMD`, `taskset -ac LIST CMD`, `watch -bt`) is NOT an exact-token match; decompose every single-dash cluster into its letters. (2) `--` END-OF-OPTIONS — model it explicitly: for a wrapper whose command follows (`watch -- CMD`, `taskset -- MASK CMD`) `--` means \"rest is the command\" → must still resolve/deny; for a flag-that-takes-a-string (`sh -- script`) it means \"rest is NOT options\". (3) OPTION-REGION TERMINATION — an option-scan hunting a flag (e.g. shell `-c`) must STOP at `--` and at the first non-option positional, else a later positional that looks like the flag (`bash script.sh -c CMD`, where `script.sh` is the script and `-c CMD` are its args) is mis-read → over-deny. Enumerate clustered + `--` + termination, not just short/long/=/combined.** |

**Design default:** When hand-modeling a CLI's option grammar for a security guard, enumerate each flag's complete spelling set (short, long, `--long=`, clustered) from the man page AND list the command's non-exec/print-and-exit modes — never model one spelling and assume the rest match.
