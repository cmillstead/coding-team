# C18

| ID | Pattern | Check before dispatch |
|----|---------|----------------------|
| C18 | `@tags: metric-aggregate; reasoning-shape; scope:both` **A metric DEFINED at one aggregation granularity (file/module/user) but IMPLEMENTED by matching/keying at a FINER granularity (exact symbol/line/row) silently mis-aggregates: the coarse rollup is STARVED whenever the fine unit that survived (post filter/dedup/budget) is not the one carrying the coarse unit's signal — and the error is DIRECTIONAL, usually toward the catch-all bucket, which is often the author's PRIOR (a confirmation-bias trap).** Same-model audits read the code with the assumption that wrote it ("match the symbol, roll up by file") and miss that the rollup is starved. Caught on the Axon anchor-provenance probe: `classify_provenance_refined` required `anchor.symbol_name == fp.symbol` before deciding a wrong FILE was anchor-reached, then `provenance_file_counts` rolled up per file by strongest provenance. But once budget/dedup runs, a wrong file's surviving FP chunk is often a DIFFERENT symbol than the anchor seed — so a file genuinely reached via an anchor on `helper_a`, surviving as `helper_b`, was classified `SemanticOnly`, mis-bucketing the file away from `WrongAnchor` toward `SemanticOnly` — inflating exactly the bucket the prior (D190) predicted dominant. Four same-model auditors (spec/simplify/harden/QA) missed it; Codex caught it. | For any metric whose REPORTED unit is coarse (file/module/package/user) but whose IMPLEMENTATION matches/keys/filters on a FINER unit (symbol/line/row/session): confirm the coarse rollup cannot be starved — would the coarse unit's signal be MISSED if the specific fine unit that survives filtering/dedup/budget is not the one carrying it? Match at the metric's OWN granularity (test FILE membership, not exact-symbol identity). Then identify the DIRECTION of any residual mis-aggregation and verify it does NOT bias toward the author's hypothesis / a catch-all bucket. Add a test where the surviving fine-unit differs from the signal-carrying fine-unit within the same coarse unit. Pairs with C17 (segment-aligned path matching for the file test) and P31 (don't infer a unit's class from its empty/partial output). |
