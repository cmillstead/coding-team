# P31

| ID | Pattern | Check before dispatch |
|----|---------|----------------------|
| P31 | `@tags: metric-aggregate; provable; scope:both` **A metric/aggregate uses a `None`/skip sentinel for "the unit produced nothing," which silently EXCLUDES legitimate zero-score MISSES from the average — inflating it — because the same sentinel also means "not applicable." Distinguish "not applicable" from "zero-score miss" with an EXPLICIT eligibility/capability flag, not a value inferred from the empty output.** When a per-unit scorer returns `Option<Score>` and the aggregate macro-averages only the `Some` values, every `None` vanishes from the denominator. That is correct for units genuinely out-of-scope (a competitor that emits no symbols → N/A), but WRONG for an in-scope unit that simply scored zero (an eligible retriever that retrieved nothing → a real recall miss). Inferring the distinction from the empty output itself (`count==0`, empty list) conflates the two: a competitor and a failed/empty eligible run look identical. Caught on the Axon symbol-precision metric: all-anchor runs, ERRORED runs (error-fallback `symbol: None`), and zero-chunk successful runs were each dropped as `N/A` instead of counted as recall=0, inflating reported recall on exactly the hardest tasks. The fix was an explicit `emits_symbols` capability flag, so an eligible unit's empty output scores as a zero-miss while a non-eligible unit stays N/A. | For any `Option`-returning per-unit metric whose aggregate skips `None`, enumerate WHY a unit can be `None` and split the reasons into "out-of-scope / not-applicable" (correctly excluded) vs "in-scope but scored zero / failed / empty" (must be COUNTED as a zero). If the code infers this from the empty output (`count==0`, empty collection, a null dependency), it conflates them — add an EXPLICIT eligibility flag carried from where capability is known (the producer/trait), and decide zero-vs-N/A on that flag. Check EVERY path that produces the empty output — success-with-nothing, error-fallback, and any "no data" short-circuit — they are siblings of the same inflation. |

**Design default:** For any metric that skips `None` outputs, add an explicit eligibility flag at the producer — never infer "not applicable" vs "zero-score miss" from the empty output alone, since success-with-nothing and not-applicable look identical.
