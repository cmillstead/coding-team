# P32

| ID | Pattern | Check before dispatch |
|----|---------|----------------------|
| P32 | `@tags: negative-existence; reasoning-shape; scope:plan` **A GROUND/plan asserts a resource or convention does NOT exist (and builds the plan around creating it FRESH / inventing a new location) based on a single empty `find`/`grep` — but it DOES exist elsewhere with an established convention to mirror.** Negative existence claims get waved through with less rigor than positive ones (P1), yet they are just as load-bearing: "there are no fixtures, so we'll make a new dir" silently invents a parallel, inconsistent location. The single search that returned empty missed a sibling directory / different path root (a `-prune` that over-excluded, the wrong subtree). Caught on the Baxter shell-classifier plan: GROUND.md asserted "No `tests/fixtures/` anywhere" (from one pruned `find`), so the plan specified `src-tauri/tests/fixtures/…` + `include_str!(concat!(env!("CARGO_MANIFEST_DIR"), …))`; in fact repo-root `tests/fixtures/fnv-vectors.json` existed, loaded by `lib.rs` via `include_str!("../../tests/fixtures/…")` — the plan should have MIRRORED that proven convention, and `CARGO_MANIFEST_DIR` would have resolved to the wrong root. | For any "X does not exist / there is no established pattern for Y, so we'll create it fresh" claim, verify the ABSENCE with the same rigor as a presence (P1): don't trust one `find`/`grep` (wrong root, over-broad `-prune`, a different dir). Confirm by asking the CONSUMER — "how does a SIBLING test/module already load this kind of resource?" Grep for the capability (`include_str!`, fixture loaders, the file extension) across the WHOLE repo, not one subtree. If a convention exists, the plan MIRRORS it (same location, same load form) instead of inventing a parallel one. A false negative-existence claim makes the plan build duplicate/inconsistent infrastructure — the inverse of P1's phantom-symbol. |
