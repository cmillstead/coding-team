# C5

| ID | Pattern | Check before dispatch |
|----|---------|----------------------|
| C5 | `@tags: test-hermeticity; provable; scope:diff` **A test that drives a LIVE external dependency (on-disk DB, real index, network, daemon) sits in the DEFAULT test suite behind only an "absent → skip" guard — so it is fast/green when the dependency is ABSENT but runs SLOW/nondeterministically (or hangs) when the dependency is PRESENT, making `cargo test`/CI non-hermetic.** The author reasons "it skips cleanly when absent," but the skip-guard only fires on the ABSENT branch; on the machine where the resource exists (the dev box, a warm CI cache), the test executes the real expensive path inside the default run. Caught on the Axon symbol-precision diff: an in-crate test called the real retriever → `AxonService::open` on the on-disk RocksDB index; it "skipped" only when the index was missing/locked, but on an indexed workspace `cargo test` sat >60s on it — and local runs that hit a held DB-lock skip path masked the hang. Fix: mark it `#[ignore = "requires a real index; run with --ignored"]` / env-gate / feature-gate so it is OPT-IN, never in the default suite — keep the absent-resource skip-guard as belt-and-suspenders for the opt-in run. | For any test that touches a live external resource (real DB/index/filesystem-state/network/daemon — not an in-memory or temp fixture), it must NOT run in the default `cargo test`/`pytest`/CI suite. An "if absent, skip" guard is insufficient: it only covers the absent branch, so the test is non-hermetic exactly where the resource exists. Gate it behind `#[ignore]` / a feature flag / an env var so the default suite is deterministic regardless of local state, and keep the skip-guard for the explicit opt-in invocation. Ask: "if this resource IS present and slow, does the default suite still finish fast and deterministically?" If not, it's not a default-suite test. (This is RUNTIME hermeticity of a test that compiles fine — beware the inverse-masking trap: a locked/absent local resource makes it skip, hiding the hang from the author.) |

**Design default:** Any test that touches a real external resource (on-disk DB, network, daemon) must be opt-in (`#[ignore]` / env-gate / feature-flag) from the start — never default-suite with an "absent→skip" guard as the sole hermeticity control.

**Partial mechanization (stays live — per-language):** A v0.5 spike (real corpus: 105 Rust files / 984 test-fns, 130 Python files / 2550 test-fns) measured detection per language. **Rust → BLOCKING** (`git-safety-guard.py` staged-index scan): FP=0 on the real corpus and on off-idiom traps, recall demonstrated on 4 real `#[ignore]` gold positives; residual FN (2, off-idiom) is fail-open-safe. It blocks *detected* ungated opens against external resources — it does **not** guarantee hermeticity, and on any unrecognized/ambiguous gate form it assumes "gated" and does not block (see the conservative-gate rule). **Python → ADVISORY only** (`write-guard.py` PreToolUse Edit|Write): the suite is hermetic-by-mandate (CLAUDE.md requires real temp fixtures), so `tmp_path`/`tempfile` are the FP surface, not a violation surface; there are no real positives, so blocking recall is unprovable (negative-reasoning-bound, the C17 shape). The full thesis above — Python, and structural/external-vs-hermetic discrimination beyond the syntactic signal — remains owned by **Codex review, by design.** C5 therefore does **not** graduate or relocate; it stays live with this truthful per-language marker.
The block fires only on a `git commit` the `is_commit` guard recognizes (not `git push`; `git -c k=v commit` / clustered-flag forms are a known fail-open-safe miss). Helper-wrapped opens remain accepted fail-open-safe FNs (both languages). Inline-comment-spoofed gates are caught on the Rust block path (the shared code-only normalizer strips string-literal contents before the `//` strip, so an inline `// tempfile` no longer suppresses a real open), but remain a fail-open-safe FN on the Python advisory path (whole-line `#` stripping only) — all owned by Codex review.
